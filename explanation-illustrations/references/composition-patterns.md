# Composition patterns and the originality rule

## Base structure types

Pick one structure — do not mix too many.

### Workflow

Good for: input -> process -> output, content production, AI workflows, automation chains.

How to draw: input on the left, Blackie or a strange machine processing in the middle, output on the right, orange arrows for the main flow.

### System fragment

Good for: information sources, filters, databases, renderers, a slice of an agent system.

How to draw: only 3–5 core modules, with Blackie performing one key action.

### Before / after

Good for: chaos/order, manual/automated, scattered/converged, anxious/calm.

How to draw: chaos on the left, calm on the right, an orange arrow between them. The character can be more exaggerated.

### Role / state

Good for: user pain points, creator states, too many tools, information anxiety, stuck-to-running.

How to draw: 2–4 small states, one short label per state.

### Concept metaphor

Good for: a content factory, an information warehouse, a mental black box, a workflow machine, an automated daily report.

How to draw: one big strange object or machine, a little input, one output. Make it memorable.

### Layered method

Good for: methodology frameworks, system layers, capability stacks, content-system layers.

How to draw: stacked boxes, not a formal pyramid; Blackie hauling bricks or building beside them.

### Map / route

Good for: idea-to-launch, user journey, content handoff path, learning path.

How to draw: one winding path, a few nodes, Blackie pulling a thread or walking.

### Mini comic panels

Good for: failure-to-success, the real process, a gripe, before/after using something.

How to draw: 2–4 small scenes, each panel expressing only one action.

## Original-metaphor generation method

Invent a fresh metaphor from the current text every time — never copy an old image.

### Three steps

1. Turn the abstract concept into a physical action: get stuck, leak out, get heavier, sort, settle, ferment, open a door, fold, unpack, flow back.
2. Turn the system structure into a low-tech object: a broken machine, a cardboard box, a drawer, a pipe, a mailbox, a strange dial, a scale, a well, a ladder, an odd workstation.
3. Let Blackie carry the action: not standing beside it, but stuck inside the machine, pulling the wrong line, guarding a gate, hauling, patching, weighing, holding the ladder, taking notes, stuffing something into some strange device.

### Object pool

- Cardboard box, drawer, old machine, funnel, scale, mailbox, door, well, ladder, pipe, ball of thread, gate, turntable, black box, hole punch, pasta press, clothesline, strange workstation.
- Use only 1–2 at a time — do not pile them up.

### Blackie action pool

- Pull, carry, stuff, scoop, press, weigh, sew, cut, twist, guard, push, catch, unpack, label, recycle.
- The action must serve the core idea — do not be strange just to be strange.

## Anti-copy rule

Do not open or replicate `assets/examples/` by default. Those images are for style calibration only — line density, white space, color restraint, Blackie's temperament.

Unless the user explicitly says "use this one / replicate this composition / adapt from this case," do not directly reuse these old compositions:

- Two breakpoints on a conveyor belt
- Blackie pulling a judgment lever inside a content machine
- Blackie turning into a funnel sorting traffic/trust/conversion
- Blackie cutting the material fish
- Blackie leading a handoff path
- Blackie pulling three layers of information sources
- Three Blackies holding a megaphone / building a bridge / opening a door
- Blackie stamping a copy toolbox
- Blackie holding a sign at a common-pitfall path

Use a new metaphor even for similar topics. For example, "handoff path" need not be a route — it could be Blackie hooking the tail of content onto a doorknob; "one input, many outputs" need not be a fish — it could be Blackie pressing one paper ball into several shapes.
