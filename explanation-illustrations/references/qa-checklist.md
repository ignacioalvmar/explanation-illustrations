# QA Checklist

## Must pass

- It is 16:9 landscape.
- The background is clean white.
- Blackie is present (unless this is a formal academic figure where it is intentionally omitted).
- Blackie performs the core action, not just decoration.
- It does not replicate an old example composition — it invents a new metaphor for the current text.
- The image is absurd, creative, interesting.
- Clean and sparse; the subject is under about 60% of the canvas.
- One image tells only one core structure.
- The English annotations are few, short, and readable.
- Orange is only for the main path or arrows.
- Red is only for key points, problems, reminders, or results.
- Blue is only for supplementary notes, feedback, or system state.

## Failure signals

Regenerate or do a local edit if any of these appear:

- A top-left title like "Common Pitfalls / Workflow / System Architecture / Roadmap".
- Blackie looks like a mascot, an emoji, or a cute cartoon.
- The image looks like a PPT, course slide, or formal flowchart.
- Too many elements, arrows, or nodes.
- Text turns into long paragraphs of explanation.
- The background has paper texture, shadow, gradient, beige, or noise.
- A realistic UI screenshot or techy interface.
- Garbled spelling or unreadable labels.
- The image is too stiff, with no absurd metaphor.
- It is too similar to an old case in `assets/examples/`.

## Iteration methods

- Too plain: make Blackie the subject of the action and add a strange-but-coherent metaphor.
- Too complex: delete nodes, keep one action and 3–5 short labels.
- Too cute: emphasize deadpan, blank serious expression, not cute, not mascot.
- Too PPT: remove the title, borders, neat grid, and excess arrows; switch to a hand-drawn scene.
- Too much like an old case: keep the core idea, swap the main object and Blackie's action.
- Wrong text: prefer a local edit; if there are many errors, regenerate and reduce the number of labels.

## Delivery judgment

A high-quality image should make the reader first feel "that's a bit strange," then understand the structure within one second.

If the first impression is a tutorial page rather than an absurd product sketch on blank paper, it fails.
